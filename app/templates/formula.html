FactSet Price-Series Formula Generator

Generate clean =FDS(...) formulas to pull daily price/volume time series. Indicators are not pulled here — they are computed in Tab 3.

Explicit row-per-day grid & rolling window. Method A writes one self-contained =FDS formula per trading day (date / close / volume) — a full time series with no reliance on Excel array-spill, e.g. =FDS("9988-HK","P_PRICE(0D)"), …(0D-1D), …(0D-2D) down the sheet. Volume uses P_VOLUME_DAY. Rolling window: 0D = today / most-recent trading day, looking back N trading days (150 by default ≈ 7 months). Re-pull anytime to refresh to the latest close. 20-day ADV (USD) is computed in-app (Tab 3) from daily price × volume, not pulled.

If FactSet returns no data, check: (a) commas not colons inside the field, (b) P_VOLUME_DAY not P_VOLUME, (c) identifier format (e.g. 9988-HK, BD5CMC), (d) 0D-first order. The generator uses the universe's Symbol as-is — FactSet resolves SEDOL / exchange-ticker identifiers natively, so A-share identifiers like BD5CMC are not mangled.

Method A — Spilling (default, fast): each ticker has its OWN sheet with the ticker in A2 and the spill formulas in B2:D2 — B2 date =FDS(A2,"JULIAN(P_PRICE(0,-109D,D).dates)"), C2 close =FDS(A2,"P_PRICE(0,-109D,D)"), D2 volume =FDS(A2,"P_VOLUME_DAY(0,-109D,D)"). The formulas are stored as text so Excel can't add @ on open; run the bundled one-click ActivateSpills macro (in the Instructions sheet) once to turn them into live spilling formulas. ~3 calls per ticker. Explicit row-per-day grid (no spill, robust): one =FDS formula per trading day for close / volume — a full series always returns even without spill. Method B: put ticker in A2; column C holds rolling 0D-Nd price formulas, column D P_VOLUME_DAY volume formulas, column E explicit-date formulas (dates in B).
